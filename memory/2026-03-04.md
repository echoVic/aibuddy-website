### PR #2: fix/windows-crlf-edit-matching (Fixes #2058)
- **问题**: Windows 上编辑失败，模型总是用 write_file 重写整个文件
- **原因**: 文件内容被规范化为 LF，但模型提供的 old_string 仍有 CRLF
- **修复**: 对 old_string 和 new_string 进行 CRLF → LF 规范化
- **测试**: ✅ 53/54 测试通过（1个因 root 权限失败，非代码问题）

### PR #3: fix/windows-frontmatter-parsing (Fixes #2053)
- **问题**: Windows 上 markdown 命令文件的 frontmatter 无法解析
- **原因**: 
  - 文件使用 CRLF (\r\n) 换行符，但正则只匹配 LF
  - UTF-8 BOM 前缀干扰匹配
- **修复**:
  - 移除 UTF-8 BOM (\uFEFF)
  - 更新正则支持 CRLF: `---(?:\r?\n)`
- **测试**: ✅ 13个测试全部通过（包括3个新的 Windows 场景测试）
